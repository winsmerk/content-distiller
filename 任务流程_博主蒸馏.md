# 博主蒸馏任务流程

> 适用场景：输入一个自定义博主名称，先运行数据采集与蒸馏任务生成，再根据 `output/_过程文件/原始素材/` 中的任务文件生成最终 HTML 蒸馏报告和创作指南 Skill 文件夹。

## 任务目标

给定一个博主名称，完成以下两个阶段：

1. 执行 `run.py`，完成环境检查、数据采集、统计分析、AI 蒸馏任务生成。
2. 读取 `output/_过程文件/原始素材/` 中生成的文件，尤其是 `{博主名称}_AI蒸馏任务.md`，生成最终产物：
   - `{博主名称}_蒸馏报告.html`
   - `{博主名称}_创作指南.skill/SKILL.md`

## 前置准备

进入项目目录并激活 Python 环境：

```bash
cd /Users/lww/Desktop/project/deepdata/blogger-distiller
source .venv/bin/activate
```

建议把 TikHub Token 放到环境变量中，不要直接写入文档或提交到 Git：

```bash
export TIKHUB_API_TOKEN="你的TikHubToken"
```

## 输入参数

将下面的 `{博主名称}` 替换为你要分析的账号名称：

```text
{博主名称}
```

示例：

```text
野生大鹅
```

## Step 1：执行数据采集与蒸馏任务生成

运行：

```bash
python run.py "{博主名称}" --token "$TIKHUB_API_TOKEN"
```

运行过程中按提示选择：

```text
平台：小红书 / 抖音
模式：A 拆解对标博主 / B 诊断我的账号
采集数量：30 / 50 / 80
视频口播：首次建议跳过，后续需要更深分析时再开启
```

推荐第一次使用：

```text
模式：A
采集数量：50
视频口播：跳过
```

成功后，重点检查这个目录：

```bash
output/_过程文件/原始素材/
```

其中最关键的文件通常是：

```text
output/_过程文件/原始素材/{博主名称}_AI蒸馏任务.md
```

## Step 2：生成最终 HTML 报告和 Skill 文件夹

让 AI 执行以下任务：

```text
请根据 output/_过程文件/原始素材/{博主名称}_AI蒸馏任务.md，
并参考 output/_过程文件/原始素材/ 下同一博主的其他过程文件，
生成最终的 HTML 蒸馏报告和创作指南 Skill 文件夹。

要求：
1. HTML 输出到 output/{博主名称}_蒸馏报告.html
2. Skill 输出到 output/{博主名称}_创作指南.skill/SKILL.md
3. 严格使用 AI蒸馏任务.md 中的统计数据、结构要求、视觉规范和 Skill 模板
4. 不编造粉丝数、小红书号、发布时间等未采集到的数据
5. 生成后检查是否存在 TODO、{X}、未填占位符
```

## 最终产物

完成后应得到：

```text
output/{博主名称}_蒸馏报告.html
output/{博主名称}_创作指南.skill/SKILL.md
```

HTML 报告用于人工阅读：

```bash
open output/{博主名称}_蒸馏报告.html
```

Skill 文件夹用于安装到 AI 工具中。

安装到 Codex：

```bash
cp -R output/{博主名称}_创作指南.skill /Users/lww/.codex/skills/
```

安装后可以在新对话中这样使用：

```text
用{博主名称}的创作指南，帮我写一篇关于 XXX 的小红书笔记
```

或：

```text
按{博主名称}的内容公式，帮我生成 20 个选题
```

## 可复制的一句话任务

```text
请按项目中的“博主蒸馏任务流程”执行：
先运行 python run.py "{博主名称}" --token "$TIKHUB_API_TOKEN"，
完成交互选择和数据采集；
然后读取 output/_过程文件/原始素材/{博主名称}_AI蒸馏任务.md，
生成 output/{博主名称}_蒸馏报告.html
和 output/{博主名称}_创作指南.skill/SKILL.md。
```

## 注意事项

- 不要把 TikHub Token 写入仓库文件。
- 如果 Step 1 报 API 权限不足，去 TikHub 控制台勾选对应平台全部 API 权限。
- 如果 `output/_过程文件/原始素材/` 中没有 `{博主名称}_AI蒸馏任务.md`，说明 Step 1 没有完整跑完。
- 如果最终报告或 Skill 中出现 `{X}`、`TODO`、`待填`，说明 Step 2 需要重新补全。

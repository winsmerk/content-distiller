## 技术要求

### 设计方向：博主档案系统 / Archive Terminal

参考语言是"工业档案"——做一个"博主数据蒸馏终端"的感觉。
系统层（编号/标签/数字）用等宽字体，内容层（中文正文/标题）用衬线字体，形成双层次感。

#### 技术栈

- 单文件 HTML，所有 CSS 手写内联，**禁止使用 Tailwind CDN**
- 字体通过 Google Fonts CDN 引入（下方有具体 import）
- 三个动效全部用原生 JS 实现，无第三方库
- 折叠面板用 `<details><summary>` 原生 HTML
- 响应式，移动端断点 768px

#### 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面底色 | `#CEC9C0` | 暖石色，整体背景 |
| 主强调色 | `#8A3926` | 砖红/赤陶，模块编号色、IF→Then 左边框、反转模块背景 |
| 正文色 | `#1A1211` | 近黑，所有正文和标题 |
| 辅助/元数据色 | `#7A6E65` | 暖棕灰，字段名、标签、次要文字、边框 |
| 反转块文字色 | `#FAF6F2` | 仅在砖红底色模块内使用 |

**禁止使用**：`#FAF8F5` 米色 / 白色卡片背景 / 任何蓝紫色 / Inter/Roboto/Arial 等系统默认字体

#### 字体规则

```html
<link href="https://fonts.googleapis.com/css2?family=Space+Mono:wght@400;700&family=Noto+Serif+SC:wght@400;500;700&display=swap" rel="stylesheet">
```

- **Space Mono**：所有模块编号（01/02/...）、系统状态栏、KEY:VALUE 字段名、数据数字
- **Noto Serif SC**：所有中文正文、模块标题、段落内容
- 两者不混用：Space Mono = 系统层，Noto Serif SC = 内容层

#### 字号系统

| 层级 | 字号 | 典型元素 |
|------|------|----------|
| 系统层主体 / 折叠面板标题 | 13px | 状态栏值、summary 文字、it-kw 关键词 |
| 标签 / 元数据文字 | 11px | card-label、stat-label、dtable th、ttable th、it-label、conc-clabel、fg-label、tg-label、sys-label |
| 数据注释 / 辅助行 | 12px | hero-eyebrow、data-note、bsource、it-ev、t-meta、meta-foot、exp-body |
| 辅助正文 / 卡片次级文字 | 14px | card-val.dim、fg-body、tg-body、top10-body、bapply |
| 主正文 / 卡片主文字 | 15px | card-val、dtable 主体、it-rule、t-title、conc-list |
| 页面基础正文 / Hero 描述 | 16px | body base、hero-desc |
| 数据统计大数字 | 20px | stat-val（模块8核心数字格） |

**原则**：系统层（标签/注释）控制在 11-13px，正文层（内容/解析）控制在 14-16px，不得将两者混淆。

#### 布局系统

**顶部系统状态栏**（position: sticky, top: 0）：
- 黑底（`#0A0806`），Space Mono，字号 13px，letter-spacing 0.05em
- 格式：`SUBJECT: {博主名} | NOTES_ANALYZED: 33/62 | GENERATED: 2026-07-14 | STATUS: DISTILLED`
- STATUS 字段值用微弱 blink 动画（opacity 1→0.6 循环，2s）

**模块布局**：
- 最大宽度 1000px 居中，水平内边距 32px
- 每个模块：`display: grid; grid-template-columns: 100px 1fr; gap: 48px`
- 左列：装饰性大数字（Space Mono，约 80-96px，color: rgba(26,18,17,0.09)）
- 右列：模块标签 + 标题 + 内容
- 模块之间用 `border-bottom: 1px solid #1A1211` 分隔，**不用卡片包裹**

**反转模块**（砖红底 + 白字）：模块1、模块8、模块10
- `background: #8A3926; color: #FAF6F2`
- 通过 `margin: 0 -32px; padding: 56px 32px` 实现全宽出血效果
- 反转模块内的装饰数字：`color: rgba(250,246,242,0.15)`

**硬性约束**：
- `border-radius: 0`（全站禁止圆角）
- `box-shadow: none`（全站禁止阴影）
- 所有边框用 `1px solid #7A6E65` 或 `1px solid #1A1211`

#### 三个必须实现的动效

**1. 滚动进入 fadeInUp**
- 所有 `.module` 初始状态：`opacity: 0; transform: translateY(20px)`
- **模块1（页面顶部反转模块）**：HTML 里直接写 `class="module module-inv visible"`，页面加载时就可见，不需要 observer
- **普通模块（2-7, 9）**：`querySelectorAll('.module:not(.module-inv)')` 建 observer，进入视口加 `visible`
- **反转模块8和10**：必须单独建第二个 observer（`querySelectorAll('.module-inv:not(#mod1)')`），进入视口时同样加 `visible`，否则它们永远 opacity:0
- `.visible` 状态：`opacity: 1; transform: translateY(0); transition: opacity .6s ease, transform .6s ease`

**2. 数字 counter 动画**（仅模块8）
- 目标数字用 `data-target` 属性存储，`data-decimals` 控制小数位
- 进入视口时从 0 动画到目标值，ease-out cubic，时长 1800ms
- 需要动画的数字用实际采集到的统计数据填入，不要用示例数字占位

**3. 分割线 draw-in**
- 每个模块标题下方放一个 `.divider-wrap`，内含 `.divider-line`
- `.divider-line` 初始：`width: 0`；模块进入视口后延迟 200ms：`width: 100%; transition: width .8s ease`
- 反转模块内的分割线颜色改为 `rgba(250,246,242,0.4)`

#### 禁止事项（V1/V2/V3 已犯过的错误）

- **禁止**：内层容器背景色与页面底色相同（导致卡片消失在背景里）
- **禁止**：h2 超过 22px，正文 line-height 超过 1.8（导致内容撑满整屏）
- **禁止**：IF→Then 内容用纯 `<p>` 标签（必须用 `border-left: 4px solid #8A3926` 的容器包裹）
- **禁止**：模块标题混入"先看结论："等语气词
- **禁止**：全站只用单一品牌色（必须使用上方完整五色系统）
- **禁止**：白色卡片 / 圆角 / box-shadow
- **禁止**：`prefers-reduced-motion` 未处理（必须加 @media 禁用所有动效）
- **禁止**：JS observer 选择器写成 `.module:not(.module-inv)` 而不给反转模块建第二个 observer（导致模块8/10永久透明）
- **禁止**：`.module-inv` 没有两个类写法——必须是 `class="module module-inv"`，不能只写 `class="module-inv"`

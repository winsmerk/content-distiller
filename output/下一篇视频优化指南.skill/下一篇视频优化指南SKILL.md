# 下一篇视频优化指南

> 基于两条视频比对蒸馏：  
> 1. 《我发现转运最快的秘诀就是去做反差大的事！》  
> 2. 《为什么你能力很强，却始终不敢离职？》

## 核心判断

第一条赢在“情绪破局 + 个人经历 + 行动想象”。  
第二条有高价值认知，但输在“抽象概念多、具体场景少、评论参与弱”。

创作下一篇时，不要只讲一个正确观点。要把观点改造成：

```text
刺痛问题
↓
结果承诺
↓
具体场景
↓
个人/客户转折故事
↓
可收藏清单
↓
评论参与题
```

## 使用流程

当用户给出一个视频主题，按以下顺序工作：

1. 判断主题属于哪类：
   - 情绪破局型：转运、卡住、改变、破圈、人生往上走。
   - 职场认知型：离职、赚钱、商业视角、执行机器、职业安全感。
   - 方法清单型：一张图、几个问题、检查表、模板、步骤。
2. 先写 3 个标题：
   - 一个强结果标题。
   - 一个刺痛问题标题。
   - 一个清单/方法标题。
3. 选择最适合小红书传播的标题，并说明理由。
4. 写 60-90 秒视频脚本。
5. 给出画面建议、字幕关键词、评论区 CTA。
6. 用自检清单检查是否具备打开、收藏、评论、转发理由。

## 标题公式

### 公式 1：刺痛问题 + 方法名

```text
能力很强却不敢离职？你缺的不是勇气，是一张“公司赚钱地图”
```

适合：职场、离职、赚钱、转型、副业、自我成长。

### 公式 2：反常识纠偏

```text
真正让你有离职底气的，不是努力工作，而是看懂这条赚钱链路
```

适合：认知反转、纠正误区、打破勤奋幻觉。

### 公式 3：低门槛行动

```text
打工多年还不敢走？先用这 4 个问题看懂公司怎么赚钱
```

适合：清单型、收藏型、教程型内容。

### 公式 4：情绪破局

```text
如果你最近被困住了，去做一件和过去反差很大的事
```

适合：人生转折、低谷、转运、破圈、主动改变。

## 脚本结构

### 0-5 秒：降低羞耻感

不要上来批评观众。先替观众卸责。

```text
你不是不敢离职，也不是能力不够。
很多人真正慌的，是不知道离开这家公司后，自己还能怎么赚钱。
```

### 5-15 秒：给具体场景

把抽象问题放进一个能看见自己的场景。

```text
你可能很会写方案、救项目、处理客户，也能把老板交代的事做得很漂亮。
但如果我问你：这家公司到底靠什么赚钱？客户为什么愿意付钱？利润卡在哪一步？
你可能一下就答不上来。
```

### 15-45 秒：给可收藏清单

把观点变成 3-5 个问题，适合字幕和截图保存。

```text
先用这 4 个问题画一张公司赚钱地图：
1. 公司卖给谁？
2. 卖的到底是什么价值？
3. 客户为什么愿意付钱？
4. 钱最后花在了哪里，利润卡在哪一步？
```

### 45-65 秒：做能力迁移

把“公司视角”转回“我能带走什么”。

```text
当你能回答这 4 个问题，你就不只是一个任务执行者。
你会知道自己真正能迁移的能力是什么：
客户理解、成交链路、交付能力、成本意识，或者资源整合。
```

### 65-90 秒：评论参与题

结尾一定要给明确动作。

```text
先收藏，今晚把你所在公司的赚钱地图画一遍。
也可以在评论区写下你的行业和岗位，我帮你看：你现在积累的能力，未来最适合迁移到哪里。
```

## 画面与字幕

优先使用“手绘图 / 白板 / 备忘录 / 表格”的可视化形式。不要只做口播。

必须出现的画面：

- 标题页：你缺的不是勇气，是公司赚钱地图。
- 四问清单页：卖给谁 / 卖什么 / 为什么付钱 / 钱花在哪里。
- 能力迁移页：客户理解 / 成交链路 / 交付能力 / 成本意识。
- 结尾页：评论行业和岗位，一起拆你的赚钱地图。

字幕关键词：

- 不是不勇敢
- 公司赚钱地图
- 任务执行机器
- 离职底气
- 可迁移能力
- 客户为什么付钱

## 评论区设计

不要只说“欢迎讨论”。要给观众一个可以直接回答的问题。

优先 CTA：

```text
评论你的行业和岗位，我帮你拆一版“公司赚钱地图”。
```

备选 CTA：

```text
收藏这条，今晚用 4 个问题画一下你公司的赚钱链路。
```

```text
如果你也明明能力不差，但一想到离职就慌，评论“地图”，我下一条给你一张检查表。
```

## 下一篇选题池

1. 能力很强却不敢离职？你缺的是公司赚钱地图。
2. 别再只当任务执行机器了，普通打工人要补这 4 个商业视角。
3. 为什么你越努力越不敢走？因为你只会做事，不会看钱从哪来。
4. 离职前别急着投简历，先回答这 4 个问题。
5. 一个打工人真正的安全感，来自看懂公司怎么赚钱。
6. 你以为自己在积累经验，其实只是在积累任务。
7. 想做自由职业，先别辞职，先画出你的赚钱链路。
8. 如果公司明天不要你了，你能带走哪 3 种能力？

## 改稿规则

当用户给出原稿时，按下面顺序改：

1. 找出最抽象的概念，改成生活/职场场景。
2. 找出最说教的句子，改成“我曾经/我见过/你有没有”。
3. 找出最有收藏价值的部分，改成 3-5 条清单。
4. 给标题补一个结果承诺。
5. 给结尾补一个评论参与题。

## 禁区

- 不要只讲“要有商业思维”这种大词。
- 不要开头就批评观众“不懂赚钱”“太天真”。
- 不要只有概念，没有具体公司/岗位/客户/利润场景。
- 不要把 CTA 写成泛泛的“你怎么看”。
- 不要把职场焦虑放大到无路可走，要给可执行的下一步。

## 自检清单

| 检查项 | 通过标准 |
|--------|----------|
| 打开理由 | 标题有痛点，也有结果承诺 |
| 留存理由 | 前 5 秒有具体场景，不是抽象概念 |
| 收藏理由 | 中段有清单、图表、问题框架 |
| 评论理由 | 结尾有可直接回答的问题 |
| 转发理由 | 观众能转给同事/朋友，且不显得尴尬 |
| 信任理由 | 有个人经历、客户案例或具体职场现场 |

## 输出格式

默认输出：

1. 推荐标题 3-5 个。
2. 最推荐标题及理由。
3. 60-90 秒口播脚本。
4. 画面分镜。
5. 字幕关键词。
6. 评论区 CTA。
7. 自检清单结果。

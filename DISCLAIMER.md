## 一、工具性质

本工具是一个**多平台内容研究辅助工具**：小红书与抖音账号数据来自 [TikHub](https://tikhub.io) 公开 REST API，YouTube 单视频由 `yt-dlp` 读取。用途仅限于：

- 个人学习与研究公开内容
- 创作者进行内容风格分析与对标
- AI 辅助创作的 prompt 工程研究

本工具**不是**：
- ❌ 小红书官方工具或授权工具
- ❌ 商业数据服务
- ❌ 爬虫对抗工具或风控绕过工具

## 二、数据来源与技术边界

### 2.1 数据获取方式

小红书与抖音通过 **TikHub 的第三方公开 REST API** 获取公开数据。这意味着：

- ✅ 不模拟浏览器登录
- ✅ 不注入 Cookie / Session
- ✅ 不破解小红书加密接口
- ✅ 不使用用户账号凭证
- ✅ 不触发小红书客户端/网页版的任何交互行为

YouTube 单视频流程通过 `yt-dlp` 下载用户指定的单条视频、字幕与元数据。默认仅适用于公开内容；`--cookies-from-browser` 只应用于使用者本来已有权访问的视频，不得用于绕过付费、地区、年龄、版权或其他访问限制。

### 2.2 数据范围

默认仅涉及**已由创作者公开发布、任何人无需登录即可访问**的内容：

- 博主的公开主页信息
- 博主已公开发布的内容（笔记 / 作品）
- 公开内容下的公开评论
- 用户明确指定的单条公开 YouTube 视频、字幕、元数据与关键帧

**本工具不获取任何私密数据**：私密账号、私信、收藏夹、浏览历史、设备信息、位置信息等，均不在处理范围内。

### 2.3 不承诺的事项

- ⚠️ 不承诺 TikHub API 的可用性、准确性、稳定性（由 TikHub 提供方负责）
- ⚠️ 不承诺小红书平台政策不变（平台随时可能调整公开数据边界）
- ⚠️ 不承诺数据完全反映真实情况（API 层可能存在字段缺失、延迟等）

## 四、评论者隐私保护

本工具在处理评论数据时，**默认启用身份脱敏**：

| 字段 | 处理方式 |
|------|---------|
| 评论者昵称 | 脱敏为"读者1 / 读者2 / 作者"（按评论顺序） |
| 评论者 userId | 不保存 |
| 评论者头像 URL | 不保存 |
| 评论者 IP 属地 | 不保存 |
| **评论正文** | **保留**（作为内容研究素材） |

**评论正文是研究资产，评论者身份是隐私风险**——这是本工具的核心设计原则。如果你需要关闭脱敏，请自行承担由此产生的全部后果。
